# Digital Grievance Redressal System (CMS)

A **Digital Grievance Redressal System** designed to help students submit, track, and resolve complaints efficiently within an educational institution.
The system ensures transparency, accountability, and faster grievance resolution through a structured workflow involving students, administrators, and departments.

---

## 📌 Problem Statement

In many educational institutions, students face difficulties reporting issues such as infrastructure problems, hostel concerns, academic queries, or administrative delays. Traditional complaint systems are often manual, slow, and lack transparency.

There is a need for a **centralized digital platform** where students can easily submit complaints and track their resolution status.

---

## 🎯 Objective

The main objectives of this system are:

* Provide a **centralized grievance submission platform**
* Enable **transparent complaint tracking**
* Improve communication between **students, admin, and departments**
* Reduce the time required for resolving issues
* Maintain **structured complaint records**

---

## ⚙️ System Workflow

### Student Workflow

1. Student logs into the system
2. Submits a grievance through the complaint form
3. Complaint is stored in the database
4. Student can track the complaint status
5. Receives updates when the complaint is resolved

### Admin Workflow

1. Admin logs into the admin dashboard
2. Views all submitted complaints
3. Assigns complaints to respective departments
4. Monitors system reports

### Department Workflow

1. Department receives assigned complaints
2. Updates complaint status (Pending → In Progress → Resolved)
3. Student receives updates about resolution
